Linux Mint 22 Cinnamon 下使用 Wallpaper Engine

Posted on 2025-01-22 Edited on 2025-01-23 In Linux , Misc

虽然 Wallpaper Engine 官方并不支持原生 Linux，但是通过 linux-wallpaperengine 这个开源项目，我们也可以在 Linux 上运行 Wallpaper Engine 并正常使用壁纸了。

解决 linux-wallpaperengine 依赖

Wallpaper Engine 本体

这个没话说，是首先要在 Steam 上购买 Wallpaper Engine，然后安装。我们主要是需要 Wallpaper Engine 在 Steam 里的文件夹。我用的是双系统，在 Windows 下我 Steam 相关的目录是 C:/Steam/steamapps

linux-wallpaperengine 依赖

依赖比较多，值得注意的是需要安装 libmpv2 而非 libmpv1（后者在一些系统上已经 deprecated）

编译、安装 linux-wallpaperengine

我们首先把项目克隆到本地

然后进入目录，创建 build 文件夹用于编译

接着进入 build 文件夹，配置 linux-wallpaperengine 项目。然后编译、安装

让 linux-wallpaperengine 能够找到壁纸文件

注意这里的壁纸文件都必须是从 Wallpaper Engine 里下载的。

我们需要在 Linux 下也安装 Steam，Linux Mint 下的话可以直接在软件管理器里下载（非常方便）

Linux 下的 Steam 的文件在 ~/.steam/steam 下，这个目录对应自己的 Steam 安装目录 C:/Steam

接下来，我们把 C:/Steam/steamapps 下的所有文件和文件夹软链接到 ~/.steam/steam/steamapps 下面。由于我是双系统，Windows 的文件系统会被挂载到 /media/OS 下（这个 OS 是因为我的 C 盘名字叫 OS）

虽然官方文档写的是将 assets 文件夹放在 linux-wallpaperengine 可执行文件同一级目录下，但是如果这里不创建软链接的话，会一直提示找不到 assets 文件夹。

另一个要注意的点是，每次都必须 cd /opt/linux-wallpaperengine 之后再 ./linux-wallpaperengine ......，否则就会提示 invalid arguments 之类的，不是很懂为什么。
